- Refresh tokens stored as SHA-256 hash in DB (never raw token)
- Access JWT: 15 min expiry | Refresh JWT: 7 days, httpOnly cookie

### 1.2 Database: Auth Tables [DONE]
**Completed:** 2026-06-19

**What was built:**
- `backend/app/models/base.py` ‚Äî `TimestampMixin` (created_at + updated_at with server_default + onupdate)
- `backend/app/models/user.py` ‚Äî `User` ORM model (SQLAlchemy 2.0 `Mapped` style)
- `backend/app/models/refresh_token.py` ‚Äî `RefreshToken` ORM model
- `backend/app/models/__init__.py` ‚Äî re-exports both models so Alembic autogenerate always sees them
- `backend/alembic/versions/001_create_auth_tables.py` ‚Äî migration: creates `users` + `refresh_tokens` tables, index on `refresh_tokens.user_id`
- `backend/alembic/env.py` ‚Äî updated to import `app.models` so metadata is populated

**Technical decisions:**
- Migration written manually (not autogenerated) because local DB credentials in `.env` didn't match; migration verified via `ScriptDirectory.walk_revisions()` ‚Äî it parses correctly
- `RefreshToken` has no `updated_at` ‚Äî tokens are write-once; revocation sets `revoked_at` only
- `users ‚Üí refresh_tokens` FK uses `ON DELETE CASCADE` to avoid orphaned tokens on hard-delete
- `refresh_tokens.user_id` has an explicit index (`idx_refresh_tokens_user_id`) for `logout-all` queries
- `User.refresh_tokens` relationship uses `lazy="noload"` ‚Äî tokens are never needed eagerly
### 1.3 Google OAuth + JWT Backend [DONE]
**Completed:** 2026-06-20

**What was built:**
- `backend/app/schemas/auth.py` ‚Äî `TokenResponse` Pydantic schema
- `backend/app/schemas/user.py` ‚Äî `UserMe` Pydantic schema
- `backend/app/services/auth.py` ‚Äî all auth business logic (upsert user, create/rotate/revoke tokens)
- `backend/app/api/v1/deps.py` ‚Äî `get_current_user` FastAPI dependency (Bearer token ‚Üí User)
- `backend/app/api/v1/routes/auth.py` ‚Äî 5 auth endpoints
- `backend/app/api/v1/routes/users.py` ‚Äî `GET /users/me`
- `backend/app/api/v1/__init__.py` ‚Äî wired auth + users routers

**Endpoints live:**
- `GET  /api/v1/auth/google` ‚Äî redirect to Google consent
- `GET  /api/v1/auth/google/callback` ‚Äî exchange code, upsert user, issue tokens
- `POST /api/v1/auth/refresh` ‚Äî rotate refresh token, return new access JWT
- `POST /api/v1/auth/logout` ‚Äî revoke current token, clear cookie
- `POST /api/v1/auth/logout-all` ‚Äî revoke all tokens for user
- `GET  /api/v1/users/me` ‚Äî return current user profile

**Technical decisions:**
- Google ID token decoded with `python-jose` (no `verify_signature`) ‚Äî safe because token was received directly from Google's token endpoint over HTTPS
- Access token returned as query param on redirect (`/auth/callback?token=...`) ‚Äî only way frontend can receive it from a browser redirect
- Refresh cookie scoped to `path=/api/v1/auth` ‚Äî cookie only sent on auth routes, not every API request
- `get_current_user` dependency checks `is_active=True` ‚Äî soft-deleted users cannot authenticate
### 1.4 Auth Frontend [DONE]
**Completed:** 2026-06-20

**What was built:**
- `frontend/app/globals.css` ‚Äî full design token system (Tailwind v4 @theme, dark palette, shimmer skeleton, flame pulse animation)
- `frontend/app/layout.tsx` ‚Äî root layout with Inter + JetBrains Mono fonts, AuthProvider wrapper
- `frontend/app/_lib/api.ts` ‚Äî fetch wrapper: attaches Bearer token, auto-refreshes on 401 (deduplicated), retries original request
- `frontend/app/_components/auth-provider.tsx` ‚Äî React context: token in memory, user profile, session restore on mount via refresh cookie
- `frontend/app/_components/sidebar.tsx` ‚Äî collapsible sidebar with nav, user avatar, logout button
- `frontend/app/(auth)/login/page.tsx` ‚Äî full-screen login page, "Continue with Google" button
- `frontend/app/(auth)/callback/page.tsx` ‚Äî reads ?token= from URL, stores in context, cleans URL, redirects to /dashboard
- `frontend/app/(dashboard)/layout.tsx` ‚Äî protected layout: checks auth, shows loading spinner, redirects to /login if no token
- `frontend/app/(dashboard)/dashboard/page.tsx` ‚Äî placeholder dashboard with welcome message
- `frontend/app/page.tsx` ‚Äî redirects / ‚Üí /dashboard

**Technical decisions:**
- Token stored in React state (memory), never localStorage ‚Äî XSS cannot read it
- Session restore on mount via `POST /auth/refresh` with httpOnly cookie ‚Äî users stay logged in across page refreshes
- `useSearchParams()` wrapped in `<Suspense>` on callback page ‚Äî required by Next.js 16 for static export
- API client uses deduplication for concurrent 401s ‚Äî only one refresh call fires even if multiple requests fail simultaneously
- Packages added: `framer-motion`, `lucide-react`
